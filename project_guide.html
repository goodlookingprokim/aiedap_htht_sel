프로젝트 개요

AIDT HTHT×SEL 프로젝트는 고등학교 1-2학년 학생들을 위한 AI Digital Textbook(AIDT)의 기술과 High-Tech × High-Touch(HTHT) 교육 모델을 통합하여 사회정서학습(SEL)을 촉진하는 교육 도구입니다.

이 프로젝트는 기존 파일들의 문제점을 개선하고, 사용자 경험을 향상시키며, GitHub에서 쉽게 배포할 수 있도록 파일 구조를 간소화하였습니다.

주요 개선 목표

  • 파일 구조 간소화 및 중복 파일 제거
  • 버튼 클릭 시 링크 깨짐 문제 해결
  • CSS와 JavaScript 코드의 HTML 태그 제거
  • 코드 최적화 및 가독성 개선
  • GitHub 배포 환경에 최적화

중복되는 파일을 제거하고 필수 파일만 유지하여 구조를 간소화했습니다:

기존 복잡한 구조에서 간결하고 명확한 파일 구조로 변경하여 관리가 용이해졌습니다.

버튼 링크 문제 해결

버튼 클릭 시 링크가 깨지는 문제를 해결했습니다:

// 탭 전환 함수 개선
function switchTab(tabId) {
    // 모든 탭 콘텐츠 숨기기
    document.querySelectorAll('.section-container').forEach(container => {
        container.classList.remove('active');
    });
    
    // 선택한 탭 콘텐츠 표시
    if (document.getElementById(tabId)) {
        document.getElementById(tabId).classList.add('active');
        
        // 탭 버튼 스타일 업데이트
        document.querySelectorAll('.tab-button').forEach(btn => {
            btn.classList.remove('active');
        });
        
        document.querySelector(`.tab-button[data-target="${tabId}"]`)?.classList.add('active');
        
        // 현재 탭 상태 저장
        currentTab = tabId;
        localStorage.setItem('currentTab', tabId);
    } else {
        console.error(`Tab with ID "${tabId}" not found`);
    }
}

이벤트 리스너 개선 및 오류 처리 코드 추가로 버튼 클릭 시 안정적으로 작동합니다.

CSS 및 JS 파일 최적화

HTML 태그로 감싸진 CSS 및 JavaScript 파일을 순수 파일로 변환했습니다:

변경 전 style.css:

<!DOCTYPE html>
<html lang="ko">
<head>
    <meta charset="UTF-8">
    <style>
        /* 기본 스타일 */
        :root {
            --primary-color: #4a6fa5;
            ...
        }
    </style>
</head>

변경 후 style.css:

/* 기본 스타일 */
:root {
    --primary-color: #4a6fa5;
    --primary-dark: #345381;
    --primary-light: #6e8cb8;
    ...
}

각 파일의 형식을 올바르게 수정하여 웹 표준을 준수하고 유지 관리가 쉬워졌습니다.

파일 참조 경로 수정

HTML 파일에서 CSS 및 JavaScript 파일 참조 경로를 수정했습니다:

<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>AIDT HTHT×SEL 체험형 웹 애플리케이션</title>
    <link href="https://cdn.jsdelivr.net/npm/tailwindcss@2.2.19/dist/tailwind.min.css" rel="stylesheet">
    <link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@fortawesome/fontawesome-free@6.4.0/css/all.min.css">
    <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+KR:wght@300;400;500;700&display=swap" rel="stylesheet">
    <link rel="stylesheet" href="style.css">
    <script src="script.js" defer></script>
</head>

상대 경로를 사용하여 파일을 올바르게 참조하도록 수정했습니다. 또한 defer 속성을 추가하여 페이지 로딩 성능을 개선했습니다.

GitHub 배포 방법

1. GitHub 저장소 생성

  1. GitHub 계정에 로그인합니다.
  2. 오른쪽 상단의 "+" 아이콘을 클릭하고 "New repository"를 선택합니다.
  3. 저장소 이름(예: aidt-htht-sel)을 입력합니다.
  4. 필요에 따라 설명을 추가합니다.
  5. "Public" 옵션을 선택합니다.
  6. "Create repository" 버튼을 클릭합니다.

2. 파일 업로드

  1. 생성된 저장소 페이지에서 "uploading an existing file" 링크를 클릭합니다.
  2. 다음 파일들을 업로드합니다:
    • README.md
    • index.html
    • style.css
    • script.js
    • instructor_manual.html
  3. "Commit changes" 버튼을 클릭하여 업로드를 완료합니다.

3. GitHub Pages 설정

  1. 저장소 페이지에서 "Settings" 탭을 클릭합니다.
  2. 왼쪽 사이드바에서 "Pages"를 선택합니다.
  3. "Source" 섹션에서:
    • "Branch" 드롭다운에서 "main"을 선택합니다.
    • 폴더 드롭다운에서 "/(root)"를 선택합니다.
    • "Save" 버튼을 클릭합니다.
  4. 페이지 상단에 배포된 URL이 표시될 때까지 기다립니다 (몇 분 소요).

배포된 URL은 일반적으로 https://[사용자이름].github.io/[저장소이름] 형식입니다.

4. 배포 확인

  1. GitHub Pages에서 제공한 URL로 접속합니다.
  2. 웹 애플리케이션이 제대로 로드되는지 확인합니다.
  3. 모든 버튼과 기능이 정상적으로 작동하는지 테스트합니다.
  4. 다양한 브라우저와 기기에서 접근성을 확인합니다.

사용 방법

교사용 가이드

  1. 강사용 메뉴얼 확인

    instructor_manual.html 파일을 열어 AIDT, HTHT, SEL의 개념과 수업 적용 방법을 숙지합니다.

  2. 웹 애플리케이션 체험

    index.html을 열어 직접 체험해보고 학생들에게 적용할 활동을 선택합니다.

  3. 수업 계획 준비

    메뉴얼에 제공된 수업 계획과 활동지를 참고하여 수업을 준비합니다.

  4. 학생 안내

    학생들에게 웹 애플리케이션 URL을 공유하고 활동을 안내합니다.

  5. 평가 및 피드백

    메뉴얼에 포함된 평가 루브릭을 활용하여 학생 활동을 평가하고 피드백합니다.

학생용 가이드

  1. 웹 애플리케이션 접속

    교사가 제공한 URL로 접속하거나 로컬에서 index.html 파일을 엽니다.

  2. 개념 이해

    소개 페이지에서 AIDT, HTHT, SEL의 개념을 이해합니다.

  3. 활동 참여

    관심 있는 활동(프로젝트 기획, 감정 인식, SEL 역량 개발 등)을 선택하여 참여합니다.

  4. 결과 저장

    활동 결과를 저장하거나 교사/동료와 공유합니다.

  5. 성찰 및 피드백

    활동 후 자기 성찰을 진행하고 피드백을 교환합니다.

자주 묻는 질문

Q: 버튼 클릭이 작동하지 않을 경우 어떻게 해야 하나요?

A: 개선된 코드에서 버튼 클릭 문제를 해결했지만, 만약 문제가 발생한다면 브라우저 콘솔을 확인하세요. 웹 페이지에서 F12를 눌러 개발자 도구를 열고 Console 탭에서 오류 메시지를 확인할 수 있습니다. 가장 흔한 원인은 파일 경로 문제이므로 HTML 파일과 같은 폴더에 CSS 및 JS 파일이 있는지 확인하세요.

Q: 웹 애플리케이션에서 작성한 데이터가 저장되나요?

A: 이 웹 애플리케이션은 브라우저의 LocalStorage를 활용하여 데이터를 저장합니다. 같은 브라우저와 기기를 사용하는 한 데이터가 유지됩니다. 하지만 브라우저 데이터를 삭제하면 저장된 정보도 함께 삭제되므로, 중요한 정보는 별도로 저장하는 것을 권장합니다.

Q: GitHub Pages에 배포 후 CSS나 JS가 적용되지 않습니다.

A: GitHub Pages에서 파일 경로 문제가 발생할 수 있습니다. HTML 파일에서 CSS와 JS 파일 참조를 상대 경로로 올바르게 설정했는지 확인하세요. <link rel="stylesheet" href="style.css">와 같은 형식으로 파일 이름만 사용해야 합니다. 절대 경로를 사용할 경우 GitHub Pages 저장소 이름을 포함해야 합니다.
